# Mineradio Fork 总阶段计划

## 一、项目方向

本 fork 不重新制作一个音乐播放器，也不以增加功能数量为目标。

主要方向是：

> 保留 Mineradio 已经完成的沉浸式视觉优势，改善 Windows 系统集成、桌面歌词、日常播放体验、稳定性和后续可维护性。

本项目目前尚未确定：

- 正式名称；
- 正式 Logo；
- 正式图标；
- 正式窗口标题；
- 正式可执行文件名；
- 正式安装包身份；
- 对外品牌。

在正式品牌确定之前：

- 不让 AI Agent 擅自命名；
- 不让 AI Agent擅自设计 Logo；
- 不进行对外品牌替换；
- 不发布带有原版品牌的修改版安装包；
- 不把临时技术标识视为最终名称。

明确不做：

- 完整本地音乐库；
- 本地文件夹扫描；
- 本地歌曲标签管理；
- 在线视觉预设社区；
- 视觉预设市场；
- 自动音效增强引擎；
- React、Vue 或 TypeScript 重写；
- 一次性拆分整个 `public/index.html`；
- 为增加功能数量开发自己不会使用的功能；
- 没有明确价值的平台扩展；
- 绕过会员、版权或付费限制。

路线图不是强制清单。

每个阶段都可以：

- 暂停；
- 缩小；
- 拆分；
- 延后；
- 取消；
- 只完成其中部分子阶段。

------

## 二、通用开发流程

每个阶段采用：

```text
只读调查
→ 说明现有实现
→ 确认需求边界
→ 建立独立分支
→ 最小实现
→ 静态检查
→ 手动测试
→ 审查 diff
→ 用户确认
→ 决定是否提交
```

通用规则：

- 一个分支只处理一个明确目标；
- 不同时修改两个无关阶段；
- 不在功能分支中大规模整理代码；
- 不在修复分支中加入额外功能；
- 不在安全分支中顺便重做 UI；
- 不在视觉分支中修改音乐平台接口；
- 每个子阶段都必须能够单独测试；
- 每个子阶段都必须能够单独回退。

完成并验证的内容记录到：

```text
docs/FORK_MEMORY.md
```

只记录：

- 已实现行为；
- 验证结果；
- 关键文件；
- 关键状态和 IPC；
- 用户确认保留的体验；
- 已知限制；
- 不应回退的边界。

- 由用户明确确认；
- 明确标记为临时；

### 默认数据策略

没有明确迁移需求时：

- 不自动迁移原版设置；
- 不删除原版数据；
- 不覆盖原版缓存；
- 不修改 upstream 安装；
- 不自动复制账号状态。

### 验收标准

- fork 测试不会污染原版配置；
- fork 不提示安装 upstream 版本；
- fork 不使用 upstream 补丁；
- fork 和原版缓存互不覆盖；
- 原版运行不受 fork 影响；
- 尚未确定名称和 Logo 不阻碍后续功能开发；
- 不产生公开发布产物。

高。

可能涉及：

# Stage 4：屏幕固定桌面歌词与歌词交互

## 目标

在现有桌面歌词基础上，将其完善为接近网易云音乐桌面歌词的日常使用形态。

- 显示当前歌词；

- 支持逐字高亮；
- 支持样式调整；
- 支持多显示器；
- 重启后恢复位置。

本阶段不是重新实现桌面歌词。

必须复用：

- `public/desktop-lyrics.html`；
- 已有桌面歌词 `BrowserWindow`；
- `desktop/overlay-preload.js`；
- 已有歌词时间轴；
- 已有置顶行为；
- 已有锁定和穿透；
- 已有拖动行为；
- 已有桌面歌词状态同步；

禁止创建：

- 第二个音频元素；
- 第二套播放时钟；
- 第二套歌词状态；
- 第二个桌面歌词窗口体系。

------

## Stage 4.1：桌面歌词基础布局与窗口稳定化 ✅ 已完成

**实际分支**：`feature/desktop-lyrics-layout`

**已实现**：单行/静态双行模式，当前行 + 有效下一行，空白和立即重复过滤，左/中/右对齐，偏好持久化于 `mineradio-lyric-layout-v1`。主窗口视觉控制台分段控件。桌面歌词位置可持久化并在重启/重新启用后恢复，允许保留可恢复的部分屏外位置。解锁拖拽、锁定穿透、拖拽结束最终位置同步、主窗口移动期间的桌面歌词组合抑制已完成并由 White 验收。

**当前范围结论**：垂直双行 rolling 歌词经过实验后被确认视觉不稳定，已从当前实现完全移除并延期。当前接受的双行行为是静态当前歌词 + 下一句歌词。偶发的主窗口移动初始/拖动闪烁仍可能存在，作为已接受限制保留。

**未完成**：Stage 4.2 逐字高亮、Stage 4.3 解锁工具栏、最终视觉 polish、低功耗/性能优化、更多显示策略仍未开始或未完成。

### 下一步：Stage 4.2 逐字高亮或 Stage 4.3 解锁工具栏（均未开始）

### 模式

```text
单行：
当前歌词

双行：
当前歌词
下一句歌词
```

### 要求

- 当前句最明显；
- 下一句降低亮度；
- 当前句切换平滑；
- 不显示完整歌词列表；
- 没有下一句时布局稳定；
- 无歌词时显示明确状态；
- 纯音乐不反复刷新空文本；
- 暂停后保持当前句；
- 恢复播放继续同步；
- 切歌立即清除上一首歌词。

### 位置与交互稳定化

- 水平/垂直位置持久化；
- 记录显示器身份；
- 重启和重新启用后恢复位置；
- 允许窗口部分屏外但保留可恢复可见区域；
- 解锁后拖拽；
- 锁定后鼠标穿透；
- 拖拽结束 flush 最终位置；
- 普通歌词/播放状态更新不重置窗口几何；
- 设置中的 Y 滑块仍作为显式重定位入口；
- 主窗口移动期间临时抑制桌面歌词组合，结束后恢复；
- 不使用 hide/show 恢复桌面歌词。

### 对齐

支持：

- 左对齐；
